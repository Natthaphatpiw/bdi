# ข้อจำกัดที่ทราบ (Known Limits) — เรียงตามความเสี่ยงต่อเดโม่

> หลักการของทีม: กรรมการให้อภัยข้อจำกัดที่ยอมรับตรง ๆ แต่ไม่ให้อภัยของปลอมที่จับได้
> ทุกข้อมี "ประโยคที่ใช้ตอบ" ที่ซื่อสัตย์และพาไปข้อแข็งของระบบต่อ

## 1. ความครอบคลุมข้อมูลสถานพยาบาลจริง (เสี่ยงสูงสุด)
- **ข้อจำกัด:** facility data verify แล้วจริงหนาแน่นในโซนบางกะปิ–ลาดพร้าว–ห้วยขวาง–บางนา; เขตอื่นเป็นชุด seed สมจริงที่ปักธง `DEMO_ONLY/NEEDS_CONFIRMATION` ใน DB (UI ไม่แสดงธง) และร้านยา/คลินิกบางแห่งยังไม่ยืนยันเวลาเปิดจริง
- **พฤติกรรมระบบเมื่อชนขอบ:** เขตที่ไม่มีข้อมูล → ใช้ผลใกล้เคียง + แนะโทร 1330; จังหวัดไกล → บอกตรง ๆ ว่ารุ่นนี้ครอบคลุม กทม.+ปริมณฑล
- **ประโยคตอบกรรมการ:** "รุ่นนี้เราลงลึกให้จริงในโซนนำร่องก่อน แล้ว scale ด้วย pipeline เดิม — ทุก record มีธงสถานะการตรวจสอบใน DB เพื่อไม่หลอกตัวเอง"

## 2. ความสดของกติกาสิทธิ์
- **ข้อจำกัด:** กติกา (เช่น วงเงินทันตกรรม ปกส., เงื่อนไข 30 บาทรักษาทุกที่) อิงประกาศที่รวบรวม ณ ก.ค. 2026 — ถ้ามีประกาศใหม่หลังจากนั้น ระบบยังไม่รู้จนกว่าจะอัปเดต KG
- **สิ่งที่ป้องกันไว้:** ทุกการ์ดสิทธิ์แนบชื่อประกาศ + วันที่มีผล ผู้ใช้ตรวจย้อนได้เสมอ — สถาปัตยกรรม temporal KG ออกแบบมาเพื่อการ swap ประกาศใหม่โดยเฉพาะ
- **ประโยคตอบ:** "เราไม่ได้ฝังกติกาในโมเดล — กติกาอยู่ในกราฟพร้อมวันที่มีผล อัปเดตได้โดยไม่ต้อง retrain"

## 3. ข้อจำกัด LIFF sensor
- **ข้อจำกัด:** DeviceMotion ใน LINE in-app browser บาง iOS ไม่ยอมขอ permission → สถานีถือนิ่ง/เดินใช้ไม่ได้ในหน้านั้น; Guardian detection engine จริง (ML บน sensor stream) ยังเป็น simulation seam
- **พฤติกรรมระบบ:** การ์ด degradation + เปิด external browser + ข้ามได้ — เช็คสุขภาพทำงานครบด้วยสถานี แตะ/พิมพ์ อย่างเดียวได้ (`stations_completed` บันทึกตามจริง)
- **ประโยคตอบ:** "ตัวเก็บข้อมูลกับ pipeline features เป็นของจริงทั้งหมด — ส่วน trigger อัตโนมัติเราแสดงผ่าน simulation seam ที่ engine จริงจะเสียบแทนโดยไม่แก้ UI แม้แต่บรรทัดเดียว"

## 4. ภาษา
- **ข้อจำกัด:** รองรับไทยเป็นหลัก — อังกฤษเข้าใจเนื้อหาได้ (NLU + red flag ภาษาอังกฤษพื้นฐาน) แต่ตอบเป็นไทยพร้อมประโยคแจ้ง; ภาษาอื่น (เมียนมา/กัมพูชา — แรงงานข้ามชาติ) ยังไม่รองรับ
- **ประโยคตอบ:** "Thai-first โดยตั้งใจ เพราะ ThaiLLM คือแกนคัดกรอง — ภาษาแรงงานข้ามชาติเป็น roadmap ที่มีมูลค่าสูง"

## 5. การวัด first-byte ใน CI
- **ข้อจำกัด:** `demo:check` วัด full-turn (non-streaming) — ตัวเลข first-byte < 3s ของจริงวัดมือบนเครื่องจริงผ่าน streaming (QA-CHECKLIST ข้อ 5)

## 6. เคสที่ระบบตอบเชิงสิทธิ์ได้แต่ไม่ route สถานที่
- ต่างจังหวัดไกล, คำถามสิทธิ์เฉพาะทางลึก (เช่น เบิกอุปกรณ์เฉพาะ CSMBS) → ตอบกรอบ + ส่ง 1330/1506 — โดยออกแบบ ไม่ใช่ bug

## 7. สถานะ demo:check ณ 2026-07-24 (ต้องปิดก่อนวันจริง)
- **ผ่านยืนยันแล้ว:** static 100% · unit 89/89 · adversarial 18/18 (100%) · safety 12/12 (100%) · golden 4/4 (100%)
- **ค้าง:** matrix/paraphrase รอบยืนยันสุดท้ายหลังแก้ 2 bug (ลำดับ branch `coverage_basis` ที่เคย map "อาชีพอิสระ/ไม่ได้ส่งประกันสังคม" เป็น SSS ผิด + harness ตอบคำถาม demographic ตาม persona) — **ถูกบล็อกเพราะเครดิต Anthropic API ของแอปหมดกลางการทดสอบ** ("credit balance is too low")
- **วิธีปิด:** เติมเครดิต API แล้วรัน `npm run demo:check` รอบเดียวจบ — เกณฑ์: golden/safety/static 100%, matrix+adversarial ≥95%
- **หมายเหตุเวลา:** p95 full-turn (non-streaming, dev เครื่องทีม) วัดได้ ~13.2s สูงกว่าเป้า 12s เล็กน้อย — first-byte จริงผ่าน streaming เร็วกว่ามาก วัดมือตาม QA-CHECKLIST ข้อ 5 และพิจารณา prewarm ก่อนคิวพิช

## 8. Booth demo corner
- `/demo`, `/liff/demo` เป็นเครื่องมือบูทที่จงใจคงไว้แบบไม่มีลิงก์จาก UI หลัก (auth bypass เฉพาะ path นั้น) — ไม่กระทบผู้ใช้จริง และ static sweep ยกเว้นโฟลเดอร์นี้อย่างประกาศชัด

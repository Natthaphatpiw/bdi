# QA Checklist — กดมือบนเครื่องจริงก่อนวันเดโม่

> รันคู่กับ `npm run demo:check` (อัตโนมัติ) — ไฟล์นี้คือส่วนที่เครื่องอัตโนมัติแทนไม่ได้
> ทำครบทั้ง **iOS LINE** และ **Android LINE** ติ๊กสองคอลัมน์

| # | รายการ | iOS | Android |
|---|---|---|---|
| **Golden A — นัทเล่าแทนแม่** ||||
| 1 | เปิด LIFF → เล่าเรื่อง golden A → คำถามเพิ่ม ≤4 ข้อ ต่อรอบ | ☐ | ☐ |
| 2 | หน้า review "เราเข้าใจเคสนี้ว่า" ค่าถูก (62 / บัตรทอง / บางนา / เบาหวาน+ตาพร่า / เล่าแทน) แก้ค่าได้ | ☐ | ☐ |
| 3 | Dashboard มีการ์ดครบ + การ์ดสิทธิ์มี "30 บาทรักษาทุกที่" + แหล่งที่มา | ☐ | ☐ |
| 4 | สร้าง Case Passport ได้ ไม่มีสิทธิ์นอกเคส (เช่น ทันตกรรม 900) ปน | ☐ | ☐ |
| 5 | First byte < 3 วิ (นับจากกดส่งถึงตัวอักษรแรก) — จับเวลาจริง | ☐ | ☐ |
| **Golden B — Guardian stroke path** ||||
| 6 | กดค้างโลโก้ 3 วิ → sheet "Guardian" → เลือก "สั่น" → popup ≤2 วิ พร้อมสั่น | ☐ | ☐ |
| 7 | เลือก "แขนหรือมืออ่อนแรงข้างเดียว" → BEFAST 3 การ์ด (กล้องหน้าเปิด-ปิดได้, timer นับเลข) | ☐ | ☐ |
| 8 | ตอบ "ใช่" ที่ A → เลือก onset → Emergency Mode พร้อม onset ในสคริปต์ 1669 | ☐ | ☐ |
| 9 | ปุ่มโทรเป็น tel:1669 / tel:1646 (แตะแล้วขึ้น dialer — **ห้ามต่อสายอัตโนมัติ**) | ☐ | ☐ |
| 10 | "รถไม่ว่าง" → UCEP card + ER ≥3 เรียงระยะ + ปุ่มนำทางเปิด Google Maps | ☐ | ☐ |
| 11 | "แจ้งครอบครัว" → shareTargetPicker เปิด (หรือ fallback คัดลอก+เปิด LINE) ข้อความตรง template | ☐ | ☐ |
| 12 | ER Passport ขึ้นบล็อกแดง UCEP + onset + เบอร์ญาติ | ☐ | ☐ |
| **Permission fallbacks** ||||
| 13 | ปฏิเสธ geolocation → หน้า Emergency ยังใช้ได้ครบ (ช่องที่อยู่แทนพิกัด) | ☐ | ☐ |
| 14 | ปฏิเสธ motion permission (iOS) → การ์ด degradation + ปุ่มเปิด external browser + ข้ามได้ | ☐ | ☐ |
| 15 | เช็คสุขภาพ: กด "ไว้ภายหลัง" ที่ consent → ไม่มี permission ใดถูกขอ | ☐ | ☐ |
| **ความทนทาน** ||||
| 16 | เน็ตช้า (เปิด throttle/3G): มีสถานะ "กำลังค้นข้อมูลสิทธิ์…" ไม่ใช่จอค้าง และ error มีปุ่มลองใหม่ | ☐ | ☐ |
| 17 | Refresh กลาง flow → state คืนจาก session เดิม / เริ่มเคสใหม่ได้ชัดเจน | ☐ | ☐ |
| 18 | เล่นติดกัน 5 เคสบนเครื่องเดียว → เคสไม่ปนกัน (เช็คหน้า history) | ☐ | ☐ |
| 19 | จอ 360px (เครื่องเล็ก): ไม่มี layout แตกทุก flow ใหม่ | ☐ | ☐ |
| 20 | Console/DevTools ระหว่างเดโม่หลัก: ไม่มี error แดง | ☐ | ☐ |
| **Guardian sim hygiene** ||||
| 21 | build ที่ไม่ตั้ง NEXT_PUBLIC_GUARDIAN_SIM: กดค้างโลโก้ + ?g= ต้องเงียบสนิท | ☐ | ☐ |
| 22 | กด "ฉันสบายดี" แล้ว trigger ใหม่ → เงียบ (cooldown) · เครื่องซ้อม (sim=1) → ยิงซ้ำได้ | ☐ | ☐ |
